## 1. Общая картина

Программа делает следующее:

```
Конфигурация ➜ Пустая сетка ➜ Хуки (pre) ➜ Алгоритм генерации ➜ Хуки (post) ➜ Решение ➜ Отрисовка / Файл
```

Всё начинается с файла `config.txt`, а заканчивается красивым лабиринтом в терминале и файлом с результатом.

---

## 2. Шаг 1 — Чтение конфигурации

**Файл:** `src/config.py`

Программа читает файл `config.txt`, в котором заданы параметры лабиринта:

```ini
WIDTH=20          # ширина (количество ячеек по горизонтали)
HEIGHT=15         # высота (количество ячеек по вертикали)
ENTRY=0,0         # координаты входа
EXIT=19,14        # координаты выхода
OUTPUT_FILE=maze.txt
PERFECT=true      # «идеальный» лабиринт (ровно один путь между любыми двумя ячейками)
# SEED=42         # необязательно — для воспроизводимости
```

**Что происходит внутри:**
- Библиотека `pydantic-settings` загружает значения и **автоматически проверяет типы** (например, что `WIDTH` — это целое число).
- Если какой-то параметр неверен, программа выдаст понятную ошибку ещё до начала генерации.

## 3. Шаг 2 — Создание пустой сетки

**Файл:** `src/models/maze.py` → класс `Maze`

После чтения конфигурации создаётся объект `Maze` — двумерный массив ячеек (grid).

```
Maze(width=5, height=4) создаёт сетку 5×4:

  ┌───┬───┬───┬───┬───┐
  │   │   │   │   │   │
  ├───┼───┼───┼───┼───┤
  │   │   │   │   │   │
  ├───┼───┼───┼───┼───┤
  │   │   │   │   │   │
  ├───┼───┼───┼───┼───┤
  │   │   │   │   │   │
  └───┴───┴───┴───┴───┘
```

На этом этапе **все стены закрыты** — каждая ячейка полностью изолирована от соседей.

**Как это выглядит в коде:**

```python
self.grid = [
    [Cell(x=x, y=y) for x in range(self.width)]
    for y in range(self.height)
]
```

Это создаёт список списков, где каждая позиция — объект `Cell`.

## 4. Шаг 3 — Что такое ячейка (Cell)

**Файл:** `src/models/cell.py` → класс `Cell`

Каждая ячейка хранит:

| Поле | Тип | Значение по умолчанию | Что означает |
|------|-----|-----------------------|-------------|
| `x` | int | — | Позиция по горизонтали (столбец) |
| `y` | int | — | Позиция по вертикали (строка) |
| `walls` | int | `0b1111` (15) | Битовая маска стен |
| `visited` | bool | `False` | Была ли ячейка посещена алгоритмом |
| `blocked` | bool | `False` | Заблокирована ли ячейка |

### Как работает битовая маска стен

Стены кодируются 4 битами — по одному биту на каждое направление:

```
Бит 3   Бит 2   Бит 1   Бит 0
WEST    SOUTH   EAST    NORTH
 8       4       2       1
```

**Примеры:**

| Значение | Двоичное | Стены |
|----------|----------|-------|
| `15` | `1111` | Все стены закрыты |
| `0`  | `0000` | Все стены открыты |
| `5`  | `0101` | Юг и Север закрыты |
| `10` | `1010` | Запад и Восток закрыты |

### Как убирается стена

Когда алгоритм решает соединить две соседние ячейки, вызывается метод `remove_walls_between`:

```
До:                           После:
┌───┬───┐                    ┌───┬───┐
│ A │ B │   ─── убрать ──►   │ A   B │
└───┴───┘   стену между     └───┴───┘
```

В коде это делается через побитовые операции:

```python
# Убрать стену: сбросить нужный бит
self.walls &= ~direction
```

Например, если `A` и `B` — соседи по горизонтали (B справа от A):
- У ячейки `A` убирается стена **EAST** (восток)
- У ячейки `B` убирается стена **WEST** (запад)

## 5. Шаг 4 — Генерация лабиринта (алгоритм DFS)

**Файл:** `src/algo/dfs.py` → класс `DFSMazeGenerator`

DFS (поиск в глубину) — самый понятный алгоритм. Он работает как человек, который идёт по лабиринту и прокладывает себе путь.

### Алгоритм пошагово

**Начальное состояние:** все стены закрыты, ни одна ячейка не посещена.

```
Шаг 0: Начинаем с ячейки входа (entry). Кладём её в стек.

        ┌───┬───┬───┐
        │ S │   │   │    S = start (текущая ячейка)
        ├───┼───┼───┤    Стек: [S]
        │   │   │   │
        ├───┼───┼───┤
        │   │   │   │
        └───┴───┴───┘
```

```
Шаг 1: Смотрим соседей S. Выбираем случайного непосещённого соседа.
        Убираем стену между S и соседом. Переходим к нему.

        ┌───┬───┬───┐
        │ ✓   ● │   │    ● = текущая ячейка
        ├───┼───┼───┤    ✓ = посещена
        │   │   │   │    Стек: [S, ●]
        ├───┼───┼───┤
        │   │   │   │
        └───┴───┴───┘
```

```
Шаг 2: Повторяем — выбираем случайного непосещённого соседа.

        ┌───┬───┬───┐
        │ ✓   ✓ │   │
        ├───┼───┼───┤
        │   │ ● │   │    Стек: [S, ✓, ●]
        ├───┼───┼───┤
        │   │   │   │
        └───┴───┴───┘
```

```
Шаг N: Если у текущей ячейки нет непосещённых соседей —
        ОТКАТЫВАЕМСЯ (backtrack): убираем ячейку из стека,
        возвращаемся к предыдущей и пробуем другого соседа.
```

```
Конец: Когда стек пуст — все ячейки посещены, лабиринт готов!
```

### Тот же алгоритм в коде

```python
def _generate_dfs(self, seed=None):
    rng = Random(seed)                          # генератор случайных чисел
    stack = [self.maze.entry]                    # стек начинается с входа
    self.maze.entry.visited = True               # помечаем вход как посещённый

    while stack:                                 # пока стек не пуст
        current_cell = stack[-1]                 # берём верхнюю ячейку стека

        # Ищем непосещённых соседей
        neighbors = self._get_neighbors(current_cell, visited=False)

        if neighbors:
            # Есть куда идти — выбираем случайного соседа
            next_cell = rng.choice(neighbors)
            current_cell.remove_walls_between(next_cell)  # убираем стену
            stack.append(next_cell)                        # кладём в стек
            next_cell.visited = True                       # помечаем
        else:
            # Тупик — откатываемся
            stack.pop()
```

### Почему это работает?

DFS гарантирует, что:
1. Каждая ячейка будет посещена **ровно один раз**
2. Между любыми двумя ячейками будет **ровно один путь** (= «идеальный» лабиринт)
3. Результат — **остовное дерево** (spanning tree) графа сетки

## 6. Шаг 5 — Другие алгоритмы (кратко)

Все алгоритмы решают одну задачу: построить остовное дерево сетки. Но делают это по-разному.

### Алгоритм Прима (`src/algo/prim.py`)

Принцип: **«растём наружу»**

```
1. Начинаем с одной ячейки — помечаем её как посещённую.
2. Все её непосещённые соседи добавляются в список «фронтир» (граница).
3. Случайно выбираем ячейку из фронтира.
4. Соединяем её с любым посещённым соседом (убираем стену).
5. Добавляем новых соседей в фронтир.
6. Повторяем, пока фронтир не опустеет.
```

**Результат:** лабиринт с короткими коридорами и множеством разветвлений.

### Алгоритм Краскала (`src/algo/kruskal.py`)

Принцип: **«убираем случайные стены»**

```
1. Каждая ячейка — отдельное множество (Union-Find).
2. Собираем список ВСЕХ стен и перемешиваем их случайно.
3. Для каждой стены:
   - Если ячейки по обе стороны принадлежат разным множествам → убираем стену, объединяем множества.
   - Если одному множеству → пропускаем (иначе появится цикл).
4. Повторяем, пока все ячейки не окажутся в одном множестве.
```

**Результат:** равномерно распределённый лабиринт.

### Алгоритм Уилсона (`src/algo/wilson.py`)

Принцип: **«случайные блуждания с затиранием петель»**

```
1. Помечаем одну случайную ячейку как часть лабиринта.
2. Берём любую непосещённую ячейку.
3. Начинаем случайное блуждание, пока не наткнёмся на уже посещённую ячейку.
4. Стираем все петли (loop-erased random walk): если мы были в ячейке дважды, 
   запоминаем только последний путь.
5. Прокладываем путь — убираем стены вдоль него.
6. Повторяем с шага 2, пока все ячейки не будут посещены.
```

**Результат:** равномерный случайный лабиринт без предвзятости (unbiased).

### Сравнительная таблица

| | DFS | Прим | Краскал | Уилсон |
|---|---|---|---|---|
| Стиль коридоров | Длинные, извилистые | Короткие, ветвящиеся | Сбалансированные | Равномерные |
| Сложность | Простой | Средний | Средний | Сложный |
| Файл | `dfs.py` | `prim.py` | `kruskal.py` | `wilson.py` |

## 7. Шаг 6 — Хуки (пре- и пост-обработка)

**Файлы:** `src/hooks/`

Хуки — это функции, которые применяются **до** (`pre`) или **после** (`post`) генерации лабиринта. Они позволяют модифицировать лабиринт без изменения кода алгоритмов.

### Как работают хуки

```
      pre-хуки           Генерация           post-хуки
         │                  │                    │
         ▼                  ▼                    ▼
  ┌─────────────┐    ┌─────────────┐    ┌──────────────┐
  │  Добавить   │    │  Алгоритм   │    │  Разрушить   │
  │  паттерн 42 │ ➜  │   (DFS и    │ ➜  │  несколько   │
  │  Заблокиро- │    │   другие)   │    │  стен (break │
  │  вать зоны  │    │             │    │  perfect)    │
  └─────────────┘    └─────────────┘    └──────────────┘
```

### Паттерн «42» (`pattern_42.py`)

Обязательный элемент задания: рисует цифры **42** заблокированными ячейками в центре лабиринта.

### Заблокированные зоны (`blocked_area.py`)

Позволяет сделать некоторые ячейки «непроходимыми» — алгоритм будет обходить их.

### Разрушение идеальности (`break_perfect.py`)

Убирает случайный процент стен **после** генерации, чтобы создать альтернативные пути.

## 8. Шаг 7 — Решение лабиринта (BFS)

**Файл:** `src/solver/bfs.py` → класс `BFSMazeSolver`

После генерации лабиринта программа находит кратчайший путь от входа до выхода с помощью BFS (поиск в ширину).

### Как работает BFS

```
1. Начинаем с ячейки входа. Кладём её в очередь.
2. Берём ячейку из начала очереди.
3. Для каждого соседа, до которого можно дойти (нет стены):
   - Если сосед ещё не был посещён → добавляем в очередь, запоминаем «откуда пришли».
4. Повторяем, пока не дойдём до выхода.
5. Восстанавливаем путь, идя от выхода обратно ко входу по запомненным ячейкам.
```

### Пример

```
┌───────────┐
│ S ─ ─ ─   │       S = вход
│   ┌───┐ │ │       E = выход
│   │   │ │ │       ─ = путь решения
│   │   ─ ─ │
│   │     E │
└───────────┘
```

### Ключевой момент — проверка стен

BFS проходит **только через открытые проходы** (где стена убрана):

```python
def _get_passable_neighbors(self, cell):
    passable = []
    for (dx, dy), wall in [
        ((0, -1), Cell.WEST),
        ((1, 0),  Cell.SOUTH),
        ((0, 1),  Cell.EAST),
        ((-1, 0), Cell.NORTH)
    ]:
        neighbor = self.maze.get_cell(cell.x + dx, cell.y + dy)
        if neighbor and not cell.has_wall(wall):   # стена открыта?
            passable.append(neighbor)
    return passable
```

## 9. Шаг 8 — Отрисовка в терминале

**Файл:** `src/renderer/ascii.py` → класс `AsciiMazeRenderer`

Рендерер превращает данные лабиринта в красочный текст для терминала.

### Как сетка превращается в текст

Каждая ячейка лабиринта превращается в блок 3×3 в рендер-массиве:

```
Ячейка (1,1) без стены EAST:

  стена   стена   стена        ██ ██ ██
  стена   пусто   проход  ➜    ██      
  стена   стена   стена        ██ ██ ██
```

Рендер-массив в 2 раза больше исходной сетки плюс рамка:

```
Лабиринт 3×3 → Рендер-массив 7×7
```

### Цветовая схема

Рендерер использует библиотеку **Rich** для цветного вывода с RGB-цветами:

| Элемент | Цвет по умолчанию |
|---------|-------------------|
| Пустая ячейка | Светло-серый |
| Стена | Фиолетовый |
| Заблокированная ячейка | Серый |
| Путь решения | Жёлтый |
| Вход | Розовый |
| Выход | Красный |

## 10. Шаг 9 — Сохранение в файл

Результат записывается в файл (например, `maze.txt`) в шестнадцатеричном формате.

### Формат файла

Каждая ячейка записывается **одной шестнадцатеричной цифрой** (0–F), которая кодирует состояние стен:

```
Пример ячейки со значением стен 0b1010 (= 10 = A):
- Бит 0 (NORTH) = 0 → открыта
- Бит 1 (EAST)  = 1 → закрыта
- Бит 2 (SOUTH) = 0 → открыта
- Бит 3 (WEST)  = 1 → закрыта
```

**Пример файла:**

```
E8A6C
D159B
...

0,0
19,14
S E E S S E S S ...
```

Последние строки: координаты входа, выхода и кратчайший путь (направления N/E/S/W).

## 11. Полная схема потока данных

```
┌─────────────┐
│ config.txt  │  ← Пользователь задаёт параметры
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  Config     │  ← pydantic-settings парсит и проверяет
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ Maze(w, h)  │  ← Создаётся пустая сетка с закрытыми стенами
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ Pre-hooks   │  ← Паттерн «42», заблокированные зоны
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ Validation  │  ← Проверка: вход/выход валидны, граф связен
└──────┬──────┘
       │
       ▼
┌─────────────────┐
│  Алгоритм       │  ← DFS / Prim / Kruskal / Wilson
│  (убирает стены) │     строит остовное дерево
└──────┬──────────┘
       │
       ▼
┌─────────────┐
│ Post-hooks  │  ← Разрушение идеальности
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ BFS Solver  │  ← Находит кратчайший путь
└──────┬──────┘
       │
       ├──────────────┐
       ▼              ▼
┌────────────┐  ┌────────────┐
│  Renderer  │  │  Output    │
│  (Rich)    │  │  (hex file)│
└────────────┘  └────────────┘
```

## 12. Словарь терминов

| Термин | Объяснение |
|--------|-----------|
| **Ячейка (Cell)** | Один «квадратик» лабиринта с 4 стенами |
| **Сетка (Grid)** | Двумерный массив ячеек |
| **Стена (Wall)** | Граница между двумя соседними ячейками |
| **Битовая маска** | Число, в котором каждый бит обозначает одну стену (0 = открыта, 1 = закрыта) |
| **Стек** | Структура данных «последний пришёл — первый вышел» (LIFO) |
| **Очередь** | Структура данных «первый пришёл — первый вышел» (FIFO) |
| **DFS** | Depth-First Search — поиск в глубину |
| **BFS** | Breadth-First Search — поиск в ширину |
| **Остовное дерево** | Граф, который соединяет все вершины без циклов |
| **Perfect maze** | Лабиринт, в котором между любыми двумя ячейками ровно один путь |
| **Union-Find** | Структура данных для отслеживания множеств (используется в алгоритме Краскала) |
| **Хук (Hook)** | Функция-преобразователь, применяемая до или после генерации |
| **Фронтир** | Список ячеек на границе уже построенной части лабиринта (в алгоритме Прима) |
| **Seed** | Начальное значение для генератора случайных чисел — одинаковый seed даёт одинаковый лабиринт |

---

> 💡 **Совет:** Чтобы лучше понять, как работает алгоритм, попробуйте запустить программу с маленьким лабиринтом (например, `WIDTH=5 HEIGHT=5`) и фиксированным `SEED`. Так результат будет воспроизводимым и его легко будет разобрать вручную.
